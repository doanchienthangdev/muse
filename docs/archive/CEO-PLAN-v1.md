## Elevator Pitch

A plugin skill framework for Claude Code (+Codex/Antigravity roadmap) that lets you invoke the distinctive cognitive patterns of great thinkers. `muse:elon <text>` applies Elon's first-principles cognitive moves. `muse:all <text>` runs a curated pipeline. `muse:debate elon vs jobs <text>` surfaces real tensions. `muse:build --person=X --src=Y` lets you create personas from your own research. Every claim citation-grounded. Every persona passes an eval framework that measures distinctiveness and signature moves. NOT character cosplay — cognitive tools as reusable primitives.

After 10 sessions, the framework knows YOUR thinking patterns and surfaces personas for your blindspots. Every insight citation-backed. The docs site (muse.dev) reads like Stratechery meets Farnam Street.

## SPIKE Charter (Phase 0)

**Goal**: Prove or disprove that Claude can produce cognitively distinct outputs per persona at a level useful for technical founders.

**Success criteria**:
- Reverse Turing test: >70% of blind raters correctly identify persona from output
- A/B vs raw Claude: >60% of blind raters pick Muse output as "more useful"
- A/B vs generic system prompt: >60% of raters pick Muse output as "more distinct"
- Vietnamese test: distinctiveness holds within 10% delta vs English

**Spike deliverables**:
- 3-persona Muse MVP (no framework overhead, pure prompt scaffolding + YAML data)
- 20 benchmark prompts covering founder decisions (pricing, architecture, hiring, feature cull)
- Blind evaluation protocol + 5 human judges recruited
- Decision memo: GO / NO-GO / PIVOT based on data

**NO-GO pivot path**: If spike fails, pivot to "curated prompt gist + simple CLI" (80% of value per outside voice). Skip framework investment.

## Citation Scope Clarification (from T3)

**Rule**: Citations cover **factual claims ABOUT persona's views on analogous problems**, NOT the cognitive move frames themselves.

**Example**:
- ❌ BAD: `signature_move: "Apply first principles" → citation: "interview-2018"`
- ✅ GOOD: `on_analogous_problems: [{topic: "Battery pricing", position: "Reduced cost 8x by attacking material-level assumptions", citation: "sandy-munro-2018"}]`

Moves are **interpretive frames** (inferred, not cited).  
Facts are **cited evidence** (concrete, traceable).

Runtime display:
```
Elon applies first-principles breakdown:
"What are we ACTUALLY trying to accomplish?"

(This interpretive pattern is documented in his pricing
decisions for Tesla batteries [1], SpaceX components [2].)

[1] Sandy Munro battery interview, 2018
[2] Everyday Astronaut Starbase Tour, 2021
```

## Eng Review Additions (2026-04-15)

Plan reviewed by `/plan-eng-review`. 10 tactical issues found + resolved. Added to scope:

### Architecture additions
- **Phase 1.5 — Distribution pipeline**: GitHub Actions release workflow, install.sh with SHA256 verify, version API via GitHub Releases, target macOS + Linux, bash 3.2+ compat
- **Formal persona schema**: `personas/schema.json` (JSON Schema draft-07), validated via `check-jsonschema` at persona load + CI
- **Harvested benchmark prompts**: 10 from real founder discussions (YC office hours, HN, Twitter) + 10 universal pre-persona prompts (eliminates benchmark author bias)
- **Self-healing config**: missing fields default, unknown fields preserved, no migration scripts (zero-downtime evolution)
- **Bounded parallel eval**: `xargs -P 3` in `eval-harness.sh` (stays under Claude API rate limits)
- **Reverse Turing protocol**: `bin/muse-spike-prepare` generates seeded markdown batches, judges answer in shared doc, `bin/muse-spike-score` de-randomizes + tallies

### Code quality additions
- **Bash 3.2 compat target**: avoid bash 4+ features (assoc arrays, mapfile, `^^`), CI matrix includes 3.2 + 4.x + 5.x
- **`bin/muse-lib.sh` shared library**: muse_error, muse_warn, muse_load_persona, muse_validate_persona, muse_resolve_path, muse_log_audit, muse_install_trap — ~50 lines, sourced by all bin/ scripts
- **Signal handling + atomic-write**: trap EXIT/INT/TERM in every interactive script, writes use temp → rename pattern

### Performance additions
- **YAML parse caching**: `~/.muse/cache/parsed/<persona>-<mtime>.json`, auto-invalidated by mtime, ~50ms → 2ms per load
- **Tier-switched + diff-scoped CI eval**: Haiku for PR A/B on diff-touched personas (~$0.15/commit), Opus for main-branch full eval (~$1.20/commit), total ~$25/month
